# Probe Technology Fundamentals

Understanding the HP3070 / Keysight i3070 ICT (In-Circuit Test) system that this project targets.

## What is a Flying Probe / ICT Tester?

An **In-Circuit Tester (ICT)** is automated test equipment (ATE) used in PCBA (Printed Circuit Board Assembly) manufacturing to verify board quality after soldering. Instead of a bed-of-nails fixture, a **flying probe** tester uses motorized probe arms that physically contact test points on the board.

**Key capability**: Detects opens, shorts, wrong components, missing components, and out-of-tolerance analog values — before functional testing.

## HP3070 / Keysight i3070 Format

This project specifically targets the **Keysight Log Record Format** used by HP3070 and i3070 testers.

### Log Structure
- Each log file represents one **panel** (a manufacturing panel may contain multiple PCBs)
- Key record types:
  - `@BTEST` — Board test header (panel ID, operator, shift, line, start time)
  - `@ATEST` — Analog test results (measurements with expected/actual values)
  - `@DTEST` — Digital test results
  - `@SHORTS` — Short circuit failures
  - `@OPENS` — Open circuit failures

### Critical Fields in `@BTEST`
| Field | Description |
|-------|-------------|
| `panel_id` | Unique panel identifier |
| `board_profile_id` | Board design variant |
| `operator_id` | Who ran the test |
| `shift` | A/B/C shift (A=06:00–14:00, B=14:00–22:00, C=22:00–06:00) |
| `line_id` | Manufacturing line |
| `start_ts` | Timestamp of test start |
| `btest_status` | PASS / FAIL / ABORT |

## Key Manufacturing Concepts

### Yield
```
Yield % = (Passing boards / Total boards tested) × 100
```
- Tracked over time per board profile, operator, shift, and line
- Rolling 7-day window is the standard reporting period
- **Target**: Typically 95%+ for mature products

### Failure Pareto
- Count of failures grouped by type (shorts, opens, OOT, etc.)
- Pareto principle applies: 20% of failure modes cause 80% of defects
- **record_type** is the primary grouping key in v1

### SPC — Statistical Process Control (XmR Chart)
- **Individuals (X) chart**: Plot each board's measured value
- **Moving Range (mR) chart**: Plot the absolute difference between consecutive values
- **Control Limits**:
  - Upper Control Limit (UCL) = X̄ + 3σ
  - Lower Control Limit (LCL) = X̄ - 3σ
  - σ = MR̄ / 1.128 (Wheeler doctrine — NOT sample stdev)
- **Rule 1 (default)**: A point beyond 3σ is a signal
- **Rule 4 (default)**: A run of 8 consecutive points on one side of the center line

### Fault Profiles (in generator)
| Profile | Description | Use Case |
|---------|-------------|----------|
| `random` | Baseline random failures | Steady-state production |
| `drift` | Gradually increasing failure rate | Process degradation |
| `cluster` | Failures concentrated on specific refdes | Component batch problem |
| `process-change` | Step-change in failure rate mid-run | Process upset |

## Reference Designators (Refdes)

- `R` prefix = Resistor (e.g., R42)
- `C` prefix = Capacitor (e.g., C17)
- `U` prefix = IC / Integrated Circuit (e.g., U3)
- `L` prefix = Inductor
- `D` prefix = Diode
- `Q` prefix = Transistor
- Exact-term search matters — "C42" and "U17 pin 3" are meaningful identifiers, which is why hybrid RAG (BM25 + vector) is used over pure vector search — see [[ADRs#ADR-006]]

## IPC Standards Context

- **IPC-A-610**: Acceptability of Electronic Assemblies — defines pass/fail criteria for solder joints
- **J-STD-001**: Requirements for Soldering Electrical and Electronic Assemblies
