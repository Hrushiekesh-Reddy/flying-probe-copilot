**2. Live query against the existing sample DuckDB**

```powershell
uv run python -c @'
import duckdb
from flying_probe_copilot.analytics import yield_over_time, failure_pareto

con = duckdb.connect("data/db/sample.duckdb", read_only=True)

print("=== yield_over_time(group_by='board') ===")
for row in yield_over_time(con, group_by="board"):
    print(f"  {row.group_key:8s} total={row.total:3d} passed={row.passed:3d} yield={row.yield_pct:6.2f}% placeholder={row.placeholder_fields}")

print()
print("=== yield_over_time(group_by='shift') — placeholder data ===")
for row in yield_over_time(con, group_by="shift"):
    print(f"  shift={row.group_key} total={row.total:3d} passed={row.passed:3d} yield={row.yield_pct:6.2f}% placeholder={row.placeholder_fields}")

print()
print("=== failure_pareto(by='record_type', top_n=5) ===")
for row in failure_pareto(con, by="record_type", top_n=5):
    print(f"  {row.key:8s} count={row.count:3d} pct={row.pct_of_total:6.2f}% cum={row.cumulative_pct:6.2f}% placeholder={row.placeholder_fields}")
'@
```

Expected:
- The board-grouped output prints one or more rows, each with `placeholder=()` (no marker — board is not BUG-007-affected).
- The shift-grouped output prints exactly one row (`shift=A`) — because BUG-007 placeholder data uniformly shifts every panel to 'A' — and `placeholder=('shift',)`.
- The Pareto output prints rows in descending count order, cumulative_pct monotonically increasing, `placeholder=()`.

**You're looking for:** the placeholder marker visibility is OBVIOUS at the REPL — you can spot at a glance that the shift row is degraded data.

**3. Error message readability**

```powershell
uv run python -c @'
import duckdb
from flying_probe_copilot.analytics import yield_over_time

con = duckdb.connect(":memory:")
from flying_probe_copilot.db.schema import init_database
init_database(con)

for bad_kwargs in [
    dict(group_by="day"),
    dict(window_days=0),
    dict(window_days=-1),
]:
    try:
        yield_over_time(con, **bad_kwargs)
    except ValueError as e:
        print(f"{bad_kwargs}: ValueError: {e}")
'@
```

Expected: each line prints a ValueError with a clear, specific message naming the bad argument and what was allowed. **You're looking for:** the error tells a confused caller exactly what went wrong without dropping a stack trace.

**4. Tz-aware as_of error path**

```powershell
uv run python -c @'
import duckdb
from datetime import datetime, timezone
from flying_probe_copilot.analytics import yield_over_time
from flying_probe_copilot.db.schema import init_database

con = duckdb.connect(":memory:"); init_database(con)
try:
    yield_over_time(con, as_of=datetime(2026, 4, 14, 10, 0, tzinfo=timezone.utc))
except ValueError as e:
    print(f"OK: {e}")
'@
```

Expected: `OK: as_of must be naive UTC (tzinfo=None); received tzinfo=datetime.timezone.utc`.
