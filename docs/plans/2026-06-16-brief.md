### Open questions for owner before Step 3
1. **Branch naming.** Proposal: rename the current worktree branch to `feature/phase2-analytics-foundation` so the PR title matches the convention from prior phases (`feature/phase1b-parser`, `feature/phase1b-notebook`, etc.). Alternative: keep the harness-generated `claude/quizzical-neumann-ba99a3` name and just name the PR appropriately. **Recommendation:** rename — cheaper than relitigating convention.
2. **Return type for analytics functions.** Options:
   - **(a) Plain `list[dataclass]`** — pure stdlib, no new dependency, easy to test. Costs: callers (future Streamlit pages) will likely convert to DataFrame anyway.
   - **(b) `pandas.DataFrame`** — natural fit for downstream Streamlit + Plotly, matches the notebook idiom (`con.execute(...).df()`). Costs: adds `pandas` to `pyproject.toml` (approval-gated — requires owner sign-off).
   - **Recommendation: (a) `list[dataclass]` for v1**, with a `.to_dataframe()` convenience attached only if/when pandas is needed in the Streamlit slice. Keeps this session zero-new-deps and dependency-decision out of an analytics PR.
3. **Window semantics.** `window_days=7` proposal: anchor `as_of` to `MAX(panels.scheduled_ts)` by default (matches the notebook's deterministic-replay pattern), allow caller to override. Lower bound is `as_of - window_days`, inclusive `>=`, upper bound `<= as_of`, both UTC. **Recommendation:** lock this in the plan as the canonical window contract so every analytics function shares it.
4. **`placeholder_fields` representation.** Options:
   - **(a) Per-row `tuple[str, ...]`** — explicit, survives row-by-row filtering, easy to assert on in tests.
   - **(b) Result-set-level metadata wrapper** — `Result(rows=[...], placeholder_fields=("shift",))`. Cleaner but rows lose self-description if a caller slices them.
   - **Recommendation: (a) per-row** — survives downstream pandas/Streamlit reshaping without losing the caveat.

Resolve these BEFORE Step 3 plan-writing.
