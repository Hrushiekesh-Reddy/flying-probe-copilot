### Generator-side facts the plan relies on (locked at planning time)

From the Step 2 explore + direct re-reading of `cli.py` + `renderers/log.py`:

- **Run directory layout:** `<out>/run_YYYY-MM-DDTHH-MM-SS-ffffff/{config.yaml, manifest.json, logs/<panel_serial>.log, results.csv, results.json}`. Parser CLI input: the run directory; parser reads `manifest.json` + every file under `logs/`.
- **One log file per board:** `cli._write_outputs` line 163-168 emits one `BatchLog(boards=[board])` per panel. So each `.log` file contains exactly `1× @BATCH + 1× @BTEST + N× @BLOCK+record (+@LIM2/@LIM3)`. The parser is per-file; "batch" in the schema rolls up across files via `runs.run_id`.
- **`manifest.json` schema:** `{panel_count, fault_rate, fault_profile, seed, board_profile, failing_boards}` — does NOT include `started_at` or `run_id`. Parser derives `run_id` from the directory name (`run_<timestamp>`) and `started_at` from the earliest @BTEST `start_ts` across the run's logs.
- **Encoding:** `cp1252` + `\r\n` by default; `utf-8` + `\n` when `--encoding=utf-8` flag set on generator. No BOM either way. Parser must detect or accept both. Plan: try `utf-8` decode first; on `UnicodeDecodeError`, fall back to `cp1252` (the Log Record Format has no non-ASCII payloads, so both decode the same bytes for valid input; the fallback is defensive).
- **Float format:** `{:+.6E}` (signed mantissa, 6-digit fraction, capital E, signed 2-digit exponent). Python `float()` round-trips these losslessly into IEEE-754 doubles; round-trip tolerance covers IEEE-754 representational error only.
- **Status codes:** Numeric only — `BTESTStatus`, `AnalogStatus`, `DigitalStatus`, `ShortsStatus`, `TwoDigitStatus`, `AnalogType` IntEnums live in `generator/models.py`. Parser imports them by reference, does NOT redefine.
- **`@TJET` 2-digit format:** rendered zero-padded (`00`/`01`/`07`). Parser must accept both `"00"` and `0` shapes (it always sees `"00"` from this generator; safer to int-parse).
- **`@PF` pin list:** rendered inline `{@PIN\N|p1|p2|...}` (literal backslash before count, per `_render_pf` line 165-170). Parser tokenizer must NOT trip on the backslash. Subrecord contents ignored per OUT-OF-SCOPE.

### Schema design (locked — to be implemented in F4)

9 tables total. All `CREATE TABLE IF NOT EXISTS`. Constraints kept minimal (Phase 1b uses application-level integrity; the round-trip test is the constraint enforcer).

**Dimension tables (5):**

```sql
boards (
  board_profile_id   VARCHAR PRIMARY KEY,         -- 'small'|'medium'|'large'
  name               VARCHAR NOT NULL,
  component_count    INTEGER NOT NULL,
  net_count          INTEGER NOT NULL,
  typical_test_count INTEGER NOT NULL
)

panels (
  panel_serial       VARCHAR PRIMARY KEY,         -- e.g. 'SYN-2026W17-00042'
  board_profile_id   VARCHAR NOT NULL,            -- → boards
  panel_position     INTEGER NOT NULL,
  line_id            VARCHAR NOT NULL,
  shift              CHAR(1) NOT NULL,            -- 'A'|'B'|'C'
  scheduled_ts       TIMESTAMP NOT NULL           -- from PanelInstance.timestamp
)

operators (
  operator_id        VARCHAR PRIMARY KEY          -- e.g. 'OP-001'
)

components (
  component_id       BIGINT PRIMARY KEY,          -- surrogate sequence
  board_profile_id   VARCHAR NOT NULL,            -- → boards
  refdes             VARCHAR NOT NULL,            -- e.g. 'R12'
  component_family   VARCHAR NOT NULL,            -- 'R'|'C'|'L'|'D'|'Q'|'U'
  UNIQUE (board_profile_id, refdes)
)

tests (
  test_id            BIGINT PRIMARY KEY,          -- surrogate sequence
  board_profile_id   VARCHAR NOT NULL,            -- → boards
  block_designator   VARCHAR NOT NULL,            -- @BLOCK designator
  record_type        VARCHAR NOT NULL,            -- 'TS'|'A-RES'|'A-CAP'|...|'D-T'
  target_refdes      VARCHAR,                     -- nullable for 'TS'
  UNIQUE (board_profile_id, block_designator, record_type)
)
```

**Run-metadata dim (1):**

```sql
runs (
  run_id             VARCHAR PRIMARY KEY,         -- directory basename: 'run_2026-04-01T...'
  board_profile_id   VARCHAR NOT NULL,
  seed               INTEGER NOT NULL,
  fault_rate         DOUBLE NOT NULL,
  fault_profile      VARCHAR NOT NULL,
  panel_count        INTEGER NOT NULL,
  failing_boards     INTEGER NOT NULL,
  ingested_at        TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
)
```

**Fact tables (3):**

```sql
test_runs (
  test_run_id        BIGINT PRIMARY KEY,          -- surrogate sequence
  panel_serial       VARCHAR NOT NULL,            -- → panels
  run_id             VARCHAR NOT NULL,            -- → runs
  operator_id        VARCHAR NOT NULL,            -- → operators (from manifest/log)
  btest_status       SMALLINT NOT NULL,           -- BTESTStatus int
  start_ts           TIMESTAMP NOT NULL,
  end_ts             TIMESTAMP NOT NULL,
  duration_s         INTEGER NOT NULL,
  multiple_test      BOOLEAN NOT NULL,
  learning           BOOLEAN NOT NULL,
  known_good         BOOLEAN NOT NULL,
  board_number       INTEGER NOT NULL
)

measurements (
  measurement_id     BIGINT PRIMARY KEY,          -- surrogate sequence
  test_run_id        BIGINT NOT NULL,             -- → test_runs
  test_id            BIGINT NOT NULL,             -- → tests
  component_id       BIGINT,                      -- → components (nullable for TS)
  record_type        VARCHAR NOT NULL,
  status             SMALLINT NOT NULL,
  measured_value     DOUBLE,                      -- A-* only
  limit_high         DOUBLE,                      -- A-*  (LIM2/LIM3)
  limit_low          DOUBLE,                      -- A-*  (LIM2/LIM3)
  limit_nominal      DOUBLE,                      -- A-*  (LIM3 only)
  substatus          INTEGER,                     -- D-T 6-bit mask
  failing_vector     BIGINT,                      -- D-T
  failing_pin_count  INTEGER,                     -- D-T, TJET
  shorts_count       INTEGER,                     -- TS
  opens_count        INTEGER,                     -- TS
  phantoms_count     INTEGER,                     -- TS
  pin_count          INTEGER                      -- TJET, PF
)

failures (
  failure_id         BIGINT PRIMARY KEY,          -- surrogate sequence
  measurement_id     BIGINT NOT NULL,             -- → measurements
  test_run_id        BIGINT NOT NULL,             -- → test_runs
  panel_serial       VARCHAR NOT NULL,            -- → panels (denormalized for Pareto)
  board_profile_id   VARCHAR NOT NULL,            -- → boards (denormalized for Pareto)
  record_type        VARCHAR NOT NULL,
  status             SMALLINT NOT NULL,
  failure_category   VARCHAR NOT NULL,            -- 'SHORTS'|'ANALOG'|'DIGITAL'|'TJET'|'PIN'
  target_refdes      VARCHAR                      -- nullable for TS
)
```

Surrogate-key approach: simple Python-side counters in `ingest_run_directory` (one per surrogate-PK table). DuckDB has no autoincrement; sequences add complexity for Phase 1b. Counters are seeded at parser-CLI start and persist within a single ingest run. Re-ingesting the same run is non-idempotent in v1 (would duplicate rows) — note in `ingest.py` docstring, defer dedup to Phase 2.

**Yield query (the one S9 tests):**

```sql
-- yield by board profile over the last 7 days, anchored at MAX(start_ts) in test_runs
WITH window AS (
  SELECT MAX(start_ts) AS anchor FROM test_runs
)
SELECT
  p.board_profile_id,
  COUNT(*)                                                        AS total,
  SUM(CASE WHEN tr.btest_status = 0 THEN 1 ELSE 0 END)            AS passed,
  100.0 * SUM(CASE WHEN tr.btest_status = 0 THEN 1 ELSE 0 END)
        / COUNT(*)                                                AS yield_pct
FROM test_runs tr
JOIN panels p ON p.panel_serial = tr.panel_serial
WHERE tr.start_ts > (SELECT anchor - INTERVAL 7 DAY FROM window)
GROUP BY p.board_profile_id
ORDER BY p.board_profile_id;
```

Anchored at `MAX(start_ts)` so the test is deterministic regardless of when it runs (uses the fixture's own clock, not wall-clock). The fixture for S9 spans days [-14, 0] and seeds known-bad panels in both halves; the query must return only the last-7-day half's yield.

**#BLOCKER-4 — YYMMDDHHMMSS → TIMESTAMP conversion contract.** Generator emits `start_ts`/`end_ts` as YYMMDDHHMMSS ints (12-digit); schema declares both `TIMESTAMP NOT NULL`. The conversion layer, century rule, and parse-failure behavior were all unspecified.

**Resolution — explicit contract:**

- **Where:** in `log_parser.py`, helper `_parse_yymmddhhmmss(value: int | str) -> datetime`. Called by the `@BTEST` record parser to convert `start_ts` and `end_ts` before constructing the parsed record. `BoardTestRecord` keeps its existing `int` type on the model side; conversion is the parser→DB hand-off layer's responsibility (inside `ingest.py` when populating `test_runs.start_ts` / `end_ts`).
- **Century rule:** Python `datetime.strptime("%y%m%d%H%M%S")` semantics — pivot at year 69/70: values 00-68 → 2000-2068; 69-99 → 1969-1999. Matches the rest of the Python ecosystem and the generator's own emit (current year is 2026 = `26` → 2026). Document the pivot in the helper's docstring.
- **Parse failure:** raise `ParseError` (the existing dataclass from BLOCKER-1's `log_parser.py`); the calling `@BTEST` parser catches, appends to `ParseReport.errors`, and **skips the entire `@BTEST` record + downstream blocks for that board log file**. Other log files in the same run continue. Document in `ParseReport.errors` with line number + the offending int.
- **Schema:** `test_runs.start_ts` and `test_runs.end_ts` stay `TIMESTAMP NOT NULL` — if conversion fails, the `test_runs` row is never inserted (consistent with the "skip the whole @BTEST record" rule).

**Mandatory new tests** (folded into step 6's per-record block):
- `test_parse_yymmddhhmmss_known_value_is_2026_datetime` — input `260401083000` → `datetime(2026, 4, 1, 8, 30, 0)`. Pin the year semantics.
- `test_parse_yymmddhhmmss_year_69_is_2069_year_70_is_1970` — boundary check for pivot.
