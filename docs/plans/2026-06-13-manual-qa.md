## Test cases — run in order, note results

### Test 1 — Generator help text

```powershell
uv run generator --help
```

**Expected:** 12 flags listed: `--board-profile`, `--count`, `--out`, `--seed`, `--fault-rate`, `--fault-profile`, `--start-date`, `--end-date`, `--operators`, `--lines`, `--format`, `--encoding`. Each has a help string. Exit code 0.

### Test 2 — Smallest happy-path run

```powershell
uv run generator --board-profile=small --count=5 --seed=42 --out=data/synthetic/
```

**Expected:** A new directory under `data/synthetic/run_<timestamp>/` containing:
- `config.yaml` (the resolved config)
- `manifest.json` (panel count, fault rate, seed, etc.)
- `logs/SYN-2026W*-*.log` × 5
- `results.csv` (one row per test record across all 5 panels)
- `results.json` (structured equivalent)

**Visual inspection:** Open one of the `.log` files in a text editor (NotePad++ or VS Code). It should look something like:

```
{@BATCH|BRD-SMALL|A|1|1||ICT|BAT-...|OP-007|ICT01|TP-001|1.0|PNL-SMALL|A
{@BTEST|SYN-2026W14-00001|0|260401083000|2|all|0|N|N|260401083002|...
   {@BLOCK|shorts|0
       {@TS|0|0|0|0|shorts_test}
   }
   {@BLOCK|R12|0
       {@A-RES|0|+1.246700E+01|R12{@LIM3|+1.200000E+01|+1.320000E+01|+1.080000E+01}}
   }
   ...
}
}
```

If it looks like an INI file, or has `PASS`/`FAIL` text strings, or has bracket imbalances — that's a fail.

### Test 3 — Seed reproducibility (smoke test of automated test #57)

```powershell
uv run generator --board-profile=small --count=3 --seed=999 --out=qa_a/
uv run generator --board-profile=small --count=3 --seed=999 --out=qa_b/

# Compare any log file byte-by-byte. PowerShell:
$a = Get-Content -Raw -Encoding Byte (Get-ChildItem qa_a\run_*\logs\*.log)[0]
$b = Get-Content -Raw -Encoding Byte (Get-ChildItem qa_b\run_*\logs\*.log)[0]
Compare-Object $a $b   # expect: NO output (byte-identical)

# Cleanup
Remove-Item -Recurse -Force qa_a, qa_b
```

**Expected:** No output from `Compare-Object` (files are byte-identical).

### Test 4 — Fault injection visible

```powershell
uv run generator --board-profile=small --count=20 --seed=7 --fault-rate=0.30 --out=data/synthetic/
```

**Expected:** Of the 20 panels, ~6 (±2) should have `@BTEST` with a non-zero status (4=shorts, 6=analog, 8=digital, etc.). Quick check via PowerShell:

```powershell
$logs = Get-ChildItem (Get-ChildItem data\synthetic\run_* | Select-Object -Last 1)\logs\*.log
$fails = 0
foreach ($f in $logs) {
    $content = Get-Content -Raw $f
    if ($content -match '\{@BTEST\|[^|]+\|([1-9]\d*)\|') {
        $fails++
        Write-Host "$($f.Name): @BTEST status = $($Matches[1])"
    }
}
Write-Host "Fails: $fails of $($logs.Count)"
```

**Expected:** `Fails: 4–8 of 20` (range accounts for sampling variance at this scale).

### Test 5 — Different board profiles produce distinct sizes

```powershell
uv run generator --board-profile=small  --count=1 --seed=1 --out=qa_size/
uv run generator --board-profile=medium --count=1 --seed=1 --out=qa_size/
uv run generator --board-profile=large  --count=1 --seed=1 --out=qa_size/

Get-ChildItem qa_size\run_*\logs\*.log | Sort-Object Length | Select-Object Name, Length

Remove-Item -Recurse -Force qa_size
```

**Expected:** Three files. Smallest is `small` profile (~few KB), medium ~3-5× larger, large ~15-20× larger.

### Test 6 — Error path: unknown profile name

### Test 7 — Encoding flag

```powershell
uv run generator --board-profile=small --count=1 --seed=1 --encoding=utf8 --out=qa_utf8/

# Look at raw bytes
$bytes = Get-Content -Raw -Encoding Byte (Get-ChildItem qa_utf8\run_*\logs\*.log)[0]
$bytes[0..40]   # eyeball: should be ASCII text, no BOM, LF (0x0A) not CRLF (0x0D 0x0A)

Remove-Item -Recurse -Force qa_utf8
```

**Expected:** UTF-8 with bare LF line endings. Default run (Test 2) used CRLF.

### Test 8 — Look at a real `results.csv`

Open `data/synthetic/run_*/results.csv` from any of the runs above in Excel or similar.

**Expected columns include:** serial, batch_id, btest_status, block_designator, record_type, status, measured, lim_nominal, lim_high, lim_low, designator (or similar).

**Visual check:** all numeric columns are numeric (not strings); status codes are integers 0-17 or 80-82; measured values are floats; designator looks like `R12`, `C7`, `U18` etc.

## Pass / fail criteria

**PASS** if:
- All 8 test cases produce expected output
- Sample `.log` file *looks* like a real i3070 log (record-oriented, numeric statuses, scientific floats)
- No bracket imbalance, no `PASS`/`FAIL` strings, no clearly fabricated text
- Sizes of small/medium/large differ by the expected ratios
- Fault rate roughly matches the requested rate

**FAIL** if:
- Any error or crash
- Sample log looks like garbage / wrong format
- Seed reproducibility breaks
- Profiles produce same-sized output
