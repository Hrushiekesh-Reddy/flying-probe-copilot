### Edge cases the plan might have missed

These are NOT in the plan's SUCCESS-WHEN list. The plan author may have judged them
out-of-scope, but they're worth surfacing so the parent can confirm in Step 6.

1. **`window_days=0`.** Not in plan. Reasonable interpretation: lower bound == upper bound ⇒ only rows at exactly `as_of` instant qualify. The plan doesn't say. Likely should raise `ValueError` or return `[]`. **Suggested test:** `test_yield_with_window_days_zero_returns_empty_or_raises` — pin whichever behaviour the Executor chooses, but lock the contract.
2. **Negative `window_days`.** Likely should `ValueError`. Plan silent. **Suggested test:** `test_yield_with_negative_window_days_raises`. Same for `failure_pareto`.
3. **`window_days` larger than the entire DB span.** Should behave identically to "include everything" — no crash, all rows returned. Plan doesn't test. **Suggested test:** `test_yield_with_window_larger_than_db_span_returns_all_rows`.
4. **`top_n=0`.** Plan silent. Reasonable interpretation: return `[]`. Could also `ValueError`. **Suggested test:** `test_pareto_with_top_n_zero_returns_empty_list_or_raises`. Pin the contract.
5. **Negative `top_n`.** Likely should `ValueError`. **Suggested test:** `test_pareto_with_negative_top_n_raises_value_error`.
6. **`top_n` larger than distinct group count.** Should return all groups, no padding, no crash. Plan covered indirectly in P-04 but not explicitly. **Suggested test:** `test_pareto_top_n_larger_than_distinct_groups_returns_all_groups`.
7. **`group_by` case-sensitivity.** `"Board"` or `"BOARD"` — plan locks the allowed set as lowercase strings (`"board"`, `"shift"`, `"operator"`, `"line"`); `"Board"` should `ValueError`. Currently covered indirectly by Y-06 but only for the value `"day"`. **Suggested test:** `test_yield_with_mixed_case_group_by_raises`.
8. **`yield_pct` precision when `total < 100`.** L10's `cumulative_pct` is a float — same concern as the notebook's `ROUND(..., 2)`. The plan doesn't specify whether analytics functions round or return raw floats. **Suggested test (or doc clarification):** confirm Y-01 isclose tolerance covers this; otherwise add `test_yield_pct_not_prematurely_rounded`. Also see ambiguity G-2 below.
9. **DuckDB SMALLINT cast of `btest_status`.** Schema says `btest_status SMALLINT NOT NULL`. NULL handling concern is moot (NOT NULL). But the analytics SQL likely uses `CASE WHEN btest_status = 0 THEN 1 ELSE 0 END`; if `btest_status` is e.g. `6` (FAIL_ANALOG), it's a fail, not a pass — already covered in Y-01 via the canonical SQL match, but worth noting that NO test directly asserts "non-zero status counts as fail" in isolation. Y-01 covers it transitively. Acceptable; flagging for completeness.
10. **as_of in the future relative to MAX(start_ts).** Caller passes `as_of=datetime(2099, 1, 1)`. Plan silent. Reasonable: returns whatever falls in `[as_of - 7 days, as_of]` — likely empty unless the future window happens to include some rows. **Suggested test:** `test_yield_with_future_as_of_returns_empty_or_subset` — pin contract.
11. **as_of as naive vs tz-aware datetime.** Brief says "UTC" but Python `datetime` can be naive or aware. DuckDB TIMESTAMP comparisons may handle these differently. Plan silent. **Suggested test:** `test_yield_with_tz_aware_as_of_matches_naive_as_of_in_utc` — or doc-pin "naive UTC only" and reject aware.
12. **Concurrent insertions during query.** Out of scope (single-writer Phase 1b/2 contract); worth flagging only — do not test.
13. **`failure_pareto` when `as_of` falls between a test_run's `start_ts` and `end_ts`.** A panel that started in-window but ended out-of-window — is its failure counted? L4 anchors on `start_ts`, so it's counted. Worth one test. **Suggested test:** `test_pareto_window_anchors_on_start_ts_not_end_ts`.
14. **Identical-second timestamps tied with as_of.** If `as_of == MAX(start_ts)` exactly and multiple rows have that timestamp, all included (inclusive upper bound). Covered indirectly by Y-07. Acceptable.
15. **`failure_pareto(by="refdes")` with all-NULL refdes.** All failures are shorts (`record_type='TS'`, `target_refdes IS NULL`). After filter, total == 0 ⇒ should return `[]`, NOT divide-by-zero on `pct_of_total`. **Suggested test:** `test_pareto_by_refdes_with_only_shorts_returns_empty_list`.
