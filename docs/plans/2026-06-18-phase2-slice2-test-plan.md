### Coverage gaps flagged by the test-generator

- Per-panel value aggregation (SPC-22): the plan says value = per-panel mean(measured_value) but the edge-case table never tests the multi-measurement-per-panel case. A naive implementation could SELECT a single row or SUM instead of AVG and pass every other test (which use one measurement per panel). Added SPC-22.
- refdes isolation (SPC-23): no edge case verifies that a different refdes in the same board/window is excluded. A missing refdes predicate in the WHERE clause would contaminate the series silently; all single-refdes fixtures would still pass. Added SPC-23.
- measured_value IS NULL handling (SPC-25): the plan's SQL sketch includes 'measured_value IS NOT NULL' but no test forces a NULL row. Shorts/digital records carry NULL measured_value; without the filter a panel's mean becomes NULL/NaN. Added SPC-25.
- rule_4 run-length boundary (SPC-06 paired with SPC-05): the plan lists rule_4 +/- but does not pin the 8-vs-9-vs-7 boundary. A 7-point negative plus an 8-point positive on the SAME construction is needed to prove L2's run length of exactly 8. Without it, an implementation using 9 (Nelson) or 7 passes a generic +/- pair.
- Anomaly window exclusion (ANOM-20): edge table covers empty/single/two-group and threshold cases but never asserts that out-of-window runs are excluded from a group's failed/total rate. Slice-1 has explicit window-boundary tests; the SPC/anomaly paths reuse _window.py but that reuse is untested here. Added SPC-24 + ANOM-20.

### Silent-pass risks

- Asserting limits merely EXIST or that lcl<=mean<=ucl without pinning sigma==MR-bar/1.128. lcl<=mean<=ucl is true for BOTH the correct MR-bar estimator and a sample-stdev estimator, so it proves nothing about S2. SPC-01 must assert the exact numeric ucl/lcl AND assert NOT-isclose to the 3*sample-stdev value, otherwise the wrong-estimator bug (the brief's #1 landmine) passes silently.
- Checking alarm_flags is non-empty without checking WHICH rule fired. A point can trip rule_1 and rule_2 simultaneously; a test that only asserts 'len(alarm_flags)>0' on a rule_4 fixture passes even if rule_4 is broken and rule_1 fired instead. Every rule test (SPC-03/05/08/11) must assert the SPECIFIC rule string is in/absent-from alarm_flags.
- Opt-in gating tested only by enabling the rule, never by the silent default. If SPC-08 (rule_2 enabled, fires) exists but SPC-07 (rule_2 pattern present, default rules, stays silent) does not, an implementation that ignores the rules tuple and always evaluates rule_2 passes. The gating negative is the load-bearing half.
- Leave-one-out 'proven' only by checking baseline_mean != value. baseline_mean can differ from the group's own value yet still be the include-self mean of all groups. ANOM-01 must assert baseline_mean == the exact peer-only mean AND != the include-self mean (0.2 vs 0.375), not just that it is 'different from value'.
- ddof asserted via a series where ddof=0 and ddof=1 coincide. stdev and pstdev are equal only in degenerate cases; ANOM-06 deliberately uses [0.1,0.2,0.3] where they differ (0.1 vs 0.0816) and pins both, so a population-std implementation fails. A fixture where they coincide would let ddof=0 pass silently.
- Constant-series / single-point edge cases asserting 'no exception' but not the limit values. EC1/EC2 must assert ucl==lcl==mean (zero-width), not merely that the call returned; an implementation that returns wide bogus limits from a 0/0 guard would pass a no-raise-only check.
- Anomaly 'flagged' positive case where baseline_std happens to be 0 (all peers identical) makes z=0 and flagged=False — a positive-flag fixture built with identical peers silently tests nothing. ANOM-03 deliberately spreads peers (0.04..0.06) so baseline_std>0 and the flag genuinely depends on the z computation.
- Ordering test that passes under group_key ASC by accident. If the severity-first fixture happens to have \|z\| already correlated with alphabetical key order, an un-changed slice-1 group_key-ASC sort passes. ANOM-08 must include a group whose alphabetical order contradicts its \|z\| rank so only a true abs(z) DESC sort passes.
- Empty/[] tests that also pass if the function raised-and-was-caught elsewhere: ensure each empty-DB/zero-match test asserts equality to [] (a list), not just falsiness, so a None return or a swallowed exception is caught.
