## The Loop at a glance

| Step | Who | What |
|------|-----|------|
| 1. Document | PARENT | Capture requirements → Session Brief doc |
| 2. Explore | SUB-AGENT | What/Why/Where/When structured map |
| 3. Plan | PARENT ONLY | Goal Contract + ordered TDD steps (never delegated) |
| 4. Verify Plan | SUB-AGENT | Adversarial red-team → concerns list |
| 5. Execute | SUB-AGENT | TDD implementation, strict guardrails, log out-of-scope bugs |
| 6. Verify Execution | SUB-AGENT | Execution report (step-by-step vs plan) |
| 7. Triple Check | PARENT ONLY | Found vs Planned vs Executed — independent |
| 8. Documentation | PARENT | Logs, git ops, spawn_task for out-of-scope bugs |
| 9. Manual QA | PARENT guided | Owner hands-on test with parent's QA script |
| 10. Feedback + Handoff | PARENT | Collect feedback, write AGENT_HANDOFF_LOG entry |

---

## Hard rules

- **Planning is parent-only.** Never delegate Step 3 to a sub-agent.
- **Triple check is parent-only.** Never delegate Step 7.
- **Out-of-scope bugs:** log immediately → spawn_task chip in Step 8. Never fix silently.
- **Commit only at Step 8.** Never mid-session.
- **Push only if owner requests.**
- **No execution before approved plan.** Steps 4 + owner "go ahead" gate Step 5.
- **TDD in Step 5:** Red → Green → Refactor per step. No implementation without a failing test.
- **External-research cache:** any subagent with web/fetch tools writes downloads to `%TEMP%\agent-research\<session>\` (Windows) or `~/.cache/agent-research/<session>/` (Unix) — never under the repo. Reports carry citations + short paraphrase only; third-party source (LGPL/GPL/proprietary) is cited by URL + commit hash, never copied. Cleanup before return is mandatory and must be stated in the report. (See BUG-001, 2026-06-13.)

---

## Fast path (doc edits, typos, single-file changes)

Skip steps 2–6. Still mandatory:
- Step 7: read what you did
- Step 8: log + commit
- Step 10: brief handoff note

---

## Out-of-scope bug rule (at every step, every agent)

Found a bug outside current scope?
1. Add to `docs/logs/BUG_LOG.md` NOW
2. Note in your output
3. Continue working — do NOT fix it
4. Parent surfaces via `spawn_task` at Step 8
