## The Loop at a glance

| Step | Who | What |
|------|-----|------|
| 1. Document | PARENT | Capture requirements → Session Brief doc |
| 2. Explore | SUB-AGENT | What/Why/Where/When structured map |
| 3. Plan | PARENT ONLY | Goal Contract + ordered TDD steps (never delegated) |
| 4. Test-Case Plan | SUB-AGENT | Behavior-level test cases (test-generator role) — what to test, not how |
| 5. Verify Plan | SUB-AGENT | Adversarial red-team of plan + test plan → concerns list |
| 6. Decision Gate | PARENT ONLY | Owner approval: Decision Index + Coverage Check + per-decision detail |
| 7. Execute | SUB-AGENT | TDD implementation, strict guardrails, log out-of-scope bugs |
| 8. Verify Execution | SUB-AGENT | Execution report (step-by-step vs plan) |
| 9. Triple Check | PARENT ONLY | Found vs Planned vs Executed — independent |
| 10. Documentation | PARENT | Logs, git ops, spawn_task for out-of-scope bugs |
| 11. Manual QA | PARENT guided | Owner hands-on test with parent's QA script |
| 12. Feedback + Handoff | PARENT | Collect feedback, write AGENT_HANDOFF_LOG entry |

---

## Hard rules

- **Planning is parent-only.** Never delegate Step 3 to a sub-agent.
- **Decision gate is parent-only.** Never delegate Step 6 — the parent runs the
  Decision Index + Coverage Check + per-decision detail itself.
- **Triple check is parent-only.** Never delegate Step 9.
- **Out-of-scope bugs:** log immediately → spawn_task chip in Step 10. Never fix silently.
- **Commit only at Step 10.** Never mid-session.
- **Push only if owner requests.**
- **No execution before the gate clears.** Step 5 (adversarial review) + Step 6 (decision
  gate, with owner sign-off) gate Step 7. No edits/commits/sub-agents until the gate completes.
- **TDD in Step 7:** Red → Green → Refactor per step. No implementation without a failing test.
- **External-research cache:** any subagent with web/fetch tools writes downloads to `%TEMP%\agent-research\<session>\` (Windows) or `~/.cache/agent-research/<session>/` (Unix) — never under the repo. Reports carry citations + short paraphrase only; third-party source (LGPL/GPL/proprietary) is cited by URL + commit hash, never copied. Cleanup before return is mandatory and must be stated in the report.

---

## Fast path (doc edits, typos, single-file changes)

Skip steps 2–8. Still mandatory:
- Step 9: read what you did
- Step 10: log + commit
- Step 12: brief handoff note

---

## Out-of-scope bug rule (at every step, every agent)

Found a bug outside current scope?
1. Add to `docs/logs/BUG_LOG.md` NOW
2. Note in your output
3. Continue working — do NOT fix it
4. Parent surfaces via `spawn_task` at Step 10
